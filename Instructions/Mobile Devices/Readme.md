## The Smartphone

The product, a physical Smartphone with its Operating System (OS) and Applications (Apps), is designed to be sold in large quantities. The goal is to schmooze the owner and fit all needs (on average). That's also the reason why the Smartphone has so many options for configuration.

The OS and Apps, which come with the Smartphone, are designed as well to keep the customer - You - for future sales. Therefore, especially the default Apps, are designed with [Vendor lock-in](https://en.wikipedia.org/wiki/Vendor_lock-in) in mind. Either you live with the Internet Services (e.g. Email Provider) of the Vendor or you look for Platform/Vendor independent solutions.

Luckily the Smartphone vendors want to sell to Companies/Enterprises as well and therefore have included certain security features and made compromises on  the lock-in aspect.

Of course this all depends on the Vendor and the price tag they put on their smartphone.

On the other hand, we are humans, with capabilities like

- Adaptation
- Tool usage and creation

If you just want to use the Smartphone as it is - out of the box ...
Goodbye and enjoy your life.

For convenience, the Smartphone uses a couple of methods over these technologies, but at the same time exposes you to the outside world.

#### Internet

##### Captive-Portal

Every time the smartphone establishes a connection to the internet, the smartphone tries to connect to the Captive-Portal from Google. Thus Google know where you are! There are indepent Captive-Portals, however switching to those cannot be done using the UI. Maybe you know someone, who could make the change for you.

Disable [Captive-Portal-Check](https://en.wikipedia.org/wiki/Captive_portal) and reduce data beeing send to Google. No longer "ᯤₓ"

```console
adb shell settings put global captive_portal_detection_enabled 0
adb shell settings put global captive_portal_server localhost
adb shell settings put global captive_portal_mode 0
```

From [Kuketz-Blog > Empfehlungsecke](https://www.kuketz-blog.de/empfehlungsecke)

##### Private Domain Name Service (DNS)

Basically for every connection, your smartphone establishes, it uses the DNS and by doing so informing the Provider of the DNS. A provider, like e.g. Mobile Provider, Google, ... , gets to know what kind of service on the Internet you are using ... including the Web Sites you visit.

- 🪜 open device "**Settings**" 
  - 🪜 select "**Network & internet**" 
    - 🪜 select "**Private DNS**" 
      - ✔️✏️ select "**Private DNS provider hostname**"
      - ✔️✏️ specify "**DNS provider hostname**" to be `dnsforge.de`
      - ✔️ tab on "**Save**"

Base on [Kuketz-Blog > Android: Werbung/Tracker schnell und einfach systemweit loswerden](https://www.kuketz-blog.de/android-werbung-tracker-schnell-und-einfach-systemweit-loswerden/)

### Location

Getting the current Location, after start using e.g. Navigation- or Map-Apps, your Smartphone uses the location of the nearest cell-phone tower to speed up the process.

#### LineageOS for microG

The microG LineageOS extension offers some options. See [GitHub: microg / UnifiedNlp](https://github.com/microg/UnifiedNlp/blob/master/README.md)

**Mozilla Location Service (MLS)**
"... network infrastructure like Bluetooth beacons, cell towers and WiFi access points. ..." from [moz://location service](https://location.services.mozilla.com/)

**Nominatim**
"Nominatim uses OpenStreetMap data to find locations on Earth by name and address (geocoding). ..." from [Nominatim](https://nominatim.org/)

- 🪜 open device "**Settings**"

      - ✏️ enable "**Mozilla Location Service**" (yes, again) 
        - ✏️ select "Allow in Settings", in the dialog "Change location access for Mozilla UnifiedNlp Back" 
          - ✏️ select "Allow all the time", in the dialog "Location permission"
          - 🛝 navigate back
      - ✏️ enable "**Mozilla Location Service**" (yes, again)
      - ✏️ enable "**Nominatim**"

By enabling both options, your current location and those locations you are interested in, will be exposed to these Web Services Provider.

# 2. Essential data

To make the device usable with all services on the Internet, you need to be able to access two essential sources.

## Sources

- User IDs and Passwords ¹ → [Password Manager](https://en.wikipedia.org/wiki/Password_manager)
- Bookmarks → Bookmark Manager (Browser vs. App)

¹ *iOS Keychain and Browser Password Manager are [lock-in solutions](https://en.wikipedia.org/wiki/Vendor_lock-in) and not the same as standalone Password Manager.  A Cloud-based password managers is an absolutely no-go.*

The challenge is, having the information available stationary (PC) **and** mobile (Smartphone/Tablet). It must be - somehow - shared between the different devices (PC, Smartphone/Tablet).

### Option A - Local - shared via USB

You manage a source either stationary **or** mobile. Basically, you store the original data  locally on a **single device** and have to __regularly__ copy backups on your other devices.

### Option B - Cloud - shared via Network

You manage a source from any of your devices. The data itself is stored somewhere on the internet and accessible by any of your devices.

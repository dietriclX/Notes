## The Smartphone

The product, a physical Smartphone with its Operating System (OS) and Applications (Apps), is designed to be sold in large quantities. The goal is to schmooze the owner and fit all needs (on average). That's also the reason why the Smartphone has so many options for configuration.

The OS and Apps, which come with the Smartphone, are designed as well to keep the customer - You - for future sales. Therefore, especially the default Apps, are designed with [Vendor lock-in](https://en.wikipedia.org/wiki/Vendor_lock-in) in mind. Either you live with the Internet Services (e.g. Email Provider) of the Vendor or you look for Platform/Vendor independent solutions.

Luckily the Smartphone vendors want to sell to Companies/Enterprises as well and therefore have included certain security features and made compromises on  the lock-in aspect.

Of course this all depends on the Vendor and the price tag they put on their smartphone.

On the other hand, we are humans, with capabilities like

- Adaptation
- Tool usage and creation

If you just want to use the Smartphone as it is - out of the box ...
Goodbye and enjoy your life.

#### Internet

##### Captive-Portal

Every time the smartphone establishes a connection to the internet, the smartphone tries to connect to the Captive-Portal from Google. Thus Google know where you are! There are independent Captive-Portals, however switching to those cannot be done using the UI. Maybe you know someone, who could make the change for you.

Disable [Captive-Portal-Check](https://en.wikipedia.org/wiki/Captive_portal) and reduce data being send to Google. No longer "ᯤₓ"

###### Android / LineageOS

Either disable this option in general

```console
adb shell settings put global captive_portal_detection_enabled 0
adb shell settings put global captive_portal_server localhost
adb shell settings put global captive_portal_mode 0
```

Or use an alternative URL/Server, instead of `connectivitycheck.gstatic.com`.

```console
adb shell 'settings put global captive_portal_http_url "http://captiveportal.kuketz.de"'
adb shell 'settings put global captive_portal_https_url "https://captiveportal.kuketz.de"'
adb shell 'settings put global captive_portal_fallback_url "http://captiveportal.kuketz.de"'
adb shell 'settings put global captive_portal_other_fallback_urls "http://captiveportal.kuketz.de"'
```

From [Kuketz-Blog > Empfehlungsecke](https://www.kuketz-blog.de/empfehlungsecke)

##### Private Domain Name Service (DNS)

Basically for every connection, your smartphone establishes, it uses the DNS and by doing so informing the Provider of the DNS. A provider, like e.g. Mobile Provider, Google, ... , gets to know what kind of service on the Internet you are using ... including the Web Sites you visit.

Alternative DNS are available, as for example from the following Provider (and there servers).

- [dnsforge.de DNS-Resolver](https://dnsforge.de/)
  - `dnsforge.de` (Variant "Normal")
  - `clean.dnsforge.de` (Variant "Clean")
  - `hard.dnsforge.de` (Variant "Hard")
  - `blank.dnsforge.de` (Variant "Blank")
- [Quad9 | A public and free DNS service for a better security and privacy](https://quad9.net/)
  - `dns.quad9.net`)

Base on [Kuketz-Blog > Android: Werbung/Tracker schnell und einfach systemweit loswerden](https://www.kuketz-blog.de/android-werbung-tracker-schnell-und-einfach-systemweit-loswerden/)

# 2. Essential data

To make the device usable with all services on the Internet, you need to be able to access two essential sources.

## Sources

- User IDs and Passwords ¹ → [Password Manager](https://en.wikipedia.org/wiki/Password_manager)
- Bookmarks → Bookmark Manager (Browser vs. App)

¹ *iOS Keychain and Browser Password Manager are [lock-in solutions](https://en.wikipedia.org/wiki/Vendor_lock-in) and not the same as standalone Password Manager.  A Cloud-based password managers is an absolutely no-go.*

### Password Manager

As soon as we use our Smartphone to access services on the Internet, we need - as on the PC - the User-IDs and Passwords to access the services. Services for example Online-Shops, Email, Forums, ...

A 3rd Party Password Manager¹ is the best option to control the database with all our User-IDs and Passwords. We are independent of the platform (PC/Smartphone), Operating System (Android/iOS/MacOS/Windows) and Web Browser (Chome/Brave/Firefox/Librewolf). In addition a Password Manager of a 3rd Party, offers additional security in the form of a separate password for accessing the User-IDs and Passwords, and/or a Key File to unlock a password database. To understand Password and Key File of a Password Manager, take a look at [KeePass Help Centre: Master Key](https://keepass.info/help/base/keys.html).

The challenge is, having the information - Password Database - available stationary (PC) **and** mobile (Smartphone/Tablet). It must be - somehow - shared between the different devices (PC, Smartphone/Tablet).

¹ See table [OS-App (Replacement/Supplement)](#os-app-replacementsupplement)

#### Option A - Local - shared via USB

You manage a source either stationary **or** mobile. Basically, you store the original data  locally on a **single device** and have to __regularly__ copy backups on your other devices.

Note: *Using a Key File, in addition to a password, is optional.*

#### Option B - Cloud - shared via Network

You manage a source from any of your devices. The data itself is stored somewhere on the internet and accessible by any of your devices.

Note: *Using a Key File, in addition to a password, recommended.*

## OS-App Uninstall

### LineageOS (for microG)

Either you disable the Apps, or remove them.

```console
# LineageOS Camera (replaced by Open Camera)
adb shell pm uninstall --user 0 org.lineageos.aperture
# LineageOS AudioFX (replaced by ???)
adb shell pm uninstall --user 0 org.lineageos.audiofx
# LineageOS Music (replaced by VLC)
adb shell pm uninstall --user 0 org.lineageos.eleven
# LineageOS Browser (replaced by Firefox, Mull, Tor Browser)
adb shell pm uninstall --user 0 org.lineageos.jelly
# LineageOS Recorder (replaced by Dolby On)
adb shell pm uninstall --user 0 org.lineageos.recorder
```
